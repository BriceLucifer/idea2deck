# Visual Direction

Use this reference after the user approves the Deck Plan.

## Define the system

Translate the approved direction into a compact visual system:

- emotional tone
- color roles rather than a large palette
- heading and body typography
- spacing rhythm and margins
- line, corner, and shape character
- image treatment and crop behavior
- chart and table treatment
- density target

Keep the system consistent without making every slide use the same composition.

## Composition

- Give each slide one dominant visual idea.
- Establish a clear reading order.
- Align elements to a small number of intentional guides.
- Use whitespace as structure.
- Avoid dense card grids unless comparison or dashboard semantics require them.
- Vary scale and composition to support pacing while preserving the same design language.

## Typography

- Use short, meaningful titles.
- Keep body text concise enough to read at presentation distance.
- Limit emphasis styles and use them consistently.
- Prevent widows, awkward wraps, and isolated short lines where practical.
- Keep source notes legible and visually subordinate, not microscopic.

## Images

- Use imagery with a communication role.
- Prefer one strong image over several weak decorative images.
- Preserve faces, products, diagrams, and important edges when cropping.
- Avoid stretching, low-resolution upscaling, repetitive stock imagery, and style drift.
- Ensure text overlays have deliberate negative space and sufficient contrast.

## Charts and tables

- Keep charts native and editable.
- Highlight the series or value that supports the slide conclusion.
- Remove unnecessary legends, gridlines, decimals, and categories.
- Use tables only when exact lookup or comparison matters.
- Split complex tables rather than shrinking them below legible size.

## Restraint

Do not add illustrations, gradients, icons, diagrams, or decorative shapes merely to make a slide look busy. A typography-led or data-led slide is valid when it communicates more clearly.
